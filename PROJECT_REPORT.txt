1. APPLICATION OVERVIEW AND TECH STACK
================================================================================
CatalogStream AI is a full-stack, containerized web application built through Vibe Coding 
that automates e-commerce listing creation for Indian marketplaces (Meesho, Amazon, Myntra).

Tech Stack:
- Frontend: HTML5, Custom Vanilla CSS3 (Glassmorphism design system, mobile touch targets, WCAG AAA high-contrast navy text), Vanilla JavaScript (ES6+, Fetch API, SSE Reader, SheetJS library).
- Backend: Python 3.11, FastAPI, Uvicorn ASGI Server, Pydantic v2, Pillow (PIL image processing).
- AI & LLM Infrastructure: LangChain, Groq Cloud API (Qwen2.5-7B-Instruct for text/batch generation, Llama 3.2 Vision for multi-modal image analysis).
- Containerization & Cloud Deployment: Docker (Python 3.11-slim base image), Render Cloud Web Service.
- Live Public URL: https://ai-powered-e-commerce-catalog-generator-uir7.onrender.com/

================================================================================
2. PROMPTING STRATEGY AND FRAMEWORKS USED
================================================================================
Frameworks:
- LangChain Expression Language (LCEL) pipelines (`prompt | model | parser`).
- Pydantic `ApparelListingSchema` for strict JSON structural enforcement:
  * product_title: (str) High-converting e-commerce product title.
  * description_bullets: (List[str]) Key product highlights.
  * seo_keywords: (List[str]) 5 high-density search terms.

Prompting Strategies:
1. System Persona & Task Formatting:
   "You are an expert e-commerce catalog strategist specializing in fashion and apparel listings. 
   Given the raw product attributes below, generate a high-converting listing.
   Raw Attributes: {raw_specs}
   {format_instructions}"

2. Multi-Modal Vision Prompt with Guardrails:
   "First, inspect the uploaded photo to verify if it contains clothing, apparel, or fashion items. 
   If it does NOT contain clothing, return a 400 Bad Request error detail. If it contains apparel, 
   extract the fabric, pattern, color, style, and generate structured listing JSON."

================================================================================
3. PHASE-BY-PHASE DEVELOPMENT SUMMARY
================================================================================
- Phase 1: Architecture Consolidation - Replaced legacy Streamlit setup with a single FastAPI server hosting both backend REST endpoints and static HTML/CSS/JS files on port 8000.
- Phase 2: Core Pipeline & Schema Engineering - Built LangChain LCEL chains for single spec transformation, batch manifest processing, and visual image extraction.
- Phase 3: Mobile & UI Redesign - Rebuilt UI with vibrant glassmorphic styling, high-contrast dark text, 48px touch targets, mobile scroll-centering tab navigation, and live preview cards for Meesho, Amazon, and Myntra.
- Phase 4: Real-Time SSE Token Streaming - Implemented `/generate-single-stream` and `/generate-image-stream` endpoints using FastAPI `StreamingResponse` and JS `ReadableStream` reader for real-time token rendering across text and photo inputs.
- Phase 5: SEO Insights Dashboard - Engineered dynamic SEO scoring algorithm (0-100) evaluating title bounds, bullet specificity, long-tail keyword quality, and indexability.
- Phase 6: Docker Containerization & Cloud Deployment - Created `Dockerfile` and `render.yaml`, secured API keys in `.env` and environment variables, and deployed live to Render.

================================================================================
5. CHALLENGES ENCOUNTERED AND HOW THEY WERE RESOLVED
================================================================================
- Challenge 1: Serving Backend and Frontend Together
  * Resolution: Mounted `/static` directory via `StaticFiles` and configured root `@app.get("/")` returning `FileResponse("static/index.html")`.
- Challenge 2: Handling Non-Clothing Image Uploads
  * Resolution: Added vision guardrails in `model/api.py` to inspect image contents and return an interactive warning alert banner when non-clothing photos are uploaded.
- Challenge 3: Real-Time Token Output
  * Resolution: Added Server-Sent Events (SSE) `/generate-single-stream` endpoint using FastAPI `StreamingResponse` and JavaScript stream reader.
- Challenge 4: Static 100% SEO Score Calculation
  * Resolution: Replaced basic threshold checks with a granular marketplace SEO scoring algorithm assessing title bounds (65-135 chars), attribute density, bullet point specificity, and long-tail phrase quality.
